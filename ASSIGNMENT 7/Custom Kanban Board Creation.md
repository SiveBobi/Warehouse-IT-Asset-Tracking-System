## Custom Kanban Board Creation

A custom Kanban board was created using GitHub Projects based on the Automated Kanban template to manage the development of the Warehouse IT Asset Tracking System.

### Board Setup

The project board was created and named:
"Warehouse IT Asset Tracking - Agile Board"

The default columns provided by the template were:
- To Do
- In Progress
- Done

To better align with real-world Agile workflows, additional columns were added:

- Testing: This column is used to verify and validate features before they are completed.
- Blocked: This column is used to highlight tasks that cannot proceed due to dependencies or issues.

The final workflow structure is:

To Do → In Progress → Testing → Blocked → Done

---

### Task Creation and Management

User stories from Assignment 6 were converted into GitHub Issues. Each issue represents a feature or functionality of the system, such as:

- Register Asset Feature
- Search Asset Function
- User Login System
- Fault Reporting System

Each issue was:
- Assigned to the user (SiveBobi)
- Labeled as "feature"
- Linked to the Kanban board

---

### Workflow Implementation

Tasks were moved across the board to represent their progress:

- Tasks in "To Do" represent work not yet started.
- Tasks in "In Progress" represent active development.
- Tasks in "Testing" are being validated.
- Tasks in "Blocked" indicate delays or dependencies.
- Tasks in "Done" represent completed work.

This movement of tasks demonstrates an active Agile workflow and helps track development progress.

---

### Benefits of Customization

The added columns improve the clarity and effectiveness of the board by:

- Ensuring quality control through the Testing stage
- Highlighting issues early using the Blocked column
- Improving visibility of task progress
- Supporting continuous delivery and Agile principles

---

### Conclusion

The customized Kanban board provides a structured and visual way to manage development tasks. It enhances productivity, improves workflow tracking, and ensures alignment with Agile software development practices.
