# Reflection: Kanban and GitHub Project Templates

This assignment involved analyzing GitHub project templates and implementing a Kanban board for managing the system development process.

One challenge was selecting the most suitable template. While multiple templates were available, each had strengths and weaknesses. The Basic Kanban template was simple but lacked automation, while the Automated Kanban provided better workflow management. Choosing the right template required understanding project needs and Agile principles.

Customizing the Kanban board also presented challenges. Deciding which columns to add required thinking about real-world development workflows. Adding columns such as "Testing" and "Blocked" improved visibility but also increased complexity.

Another challenge was translating previous assignment tasks into GitHub Issues. Each user story had to be converted into a task and properly linked to the board. This required consistency and attention to detail.

Comparing GitHub Projects to tools like Trello and Jira showed some differences. Trello is simpler and more visual, while Jira is more advanced and structured. GitHub Projects sits in between, offering good integration with code and repositories.

Overall, this assignment demonstrated the importance of visual task management in Agile development. The Kanban board provides clarity, improves workflow tracking, and ensures that tasks are completed efficiently.
