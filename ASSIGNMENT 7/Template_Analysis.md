# GitHub Project Template Analysis

## Template Comparison Table

| Template | Columns / Workflow | Automation Features | Agile Suitability |
|----------|------------------|---------------------|-------------------|
| Basic Kanban | To Do, In Progress, Done | No automation | Good for simple projects |
| Automated Kanban | To Do, In Progress, Done | Automatically moves issues when updated | Excellent for Agile workflows |
| Bug Triage | New Issues, Needs Review, Fixed | Auto-labeling and issue tracking | Best for maintenance teams |
| Team Planning | Backlog, Ready, In Progress, Done | Supports sprint planning | Good for structured Agile teams |

---

## Selected Template: Automated Kanban

### Justification

The **Automated Kanban template** was selected because it best supports Agile development for the Warehouse IT Asset Tracking System.

Reasons:

- Automatically moves tasks across columns based on status updates.
- Supports continuous workflow without manual updates.
- Aligns with Agile principles such as iterative development and continuous delivery.
- Reduces manual effort in task tracking.

This is ideal because the project includes multiple user stories, tasks, and sprint planning activities from Assignment 6.
