# Warehouse IT Asset Tracking System

## 1. Introduction

### Project Title
Warehouse IT Asset Tracking System

### Domain
Warehouse and Logistics IT Infrastructure Management

Warehouses rely heavily on IT equipment such as barcode scanners, label printers, computers, monitors, and mobile printers. Managing these devices manually can lead to poor visibility, loss of equipment, and delayed repairs.

The domain focuses on creating a centralized system that allows warehouse IT administrators to track, manage, and monitor all IT assets within the warehouse.

### Problem Statement
Currently many warehouses track equipment manually using spreadsheets or paper-based systems. This makes it difficult to know:

- Where a device is located
- Which devices are faulty
- When equipment was repaired
- Who is using the equipment

This often leads to equipment loss, downtime, and inefficient maintenance processes.

The proposed system will solve these issues by providing a centralized digital platform for managing warehouse IT assets.

### Individual Scope (Feasibility Justification)

The system will be designed and implemented as a simplified web application that includes the following features:

- Asset registration
- Asset location tracking
- Asset status management (working, faulty, under repair)
- Maintenance record tracking
- Viewing asset history

As an individual project, the implementation will focus on the core functionality and essential system components while maintaining scalability for future improvements.

The system will include:

- Web user interface
- Backend service
- Database for storing asset data
- REST API communication

This scope is feasible for a single developer over the semester while still demonstrating strong software engineering principles.
