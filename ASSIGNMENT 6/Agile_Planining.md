# Agile Planning Document
Warehouse IT Asset Tracking System

## 1. User Stories

| Story ID | User Story | Acceptance Criteria | Priority |
|----------|-----------|---------------------|----------|
| US-001 | As an IT Administrator, I want to register assets so that I can track all devices | Asset saved with unique ID | High |
| US-002 | As an IT Administrator, I want to update asset location so that I know where devices are | Location updates saved | High |
| US-003 | As an IT Administrator, I want to update asset status so that I can track faulty devices | Status updated with timestamp | High |
| US-004 | As a user, I want to search for assets so that I can find devices quickly | Results load within 2 seconds | High |
| US-005 | As a technician, I want to record maintenance so that repair history is tracked | Maintenance saved correctly | Medium |
| US-006 | As a manager, I want to view reports so that I can make decisions | Reports generated successfully | Medium |
| US-007 | As a worker, I want to report faults so that issues are resolved quickly | Fault recorded and notification sent | High |
| US-008 | As a user, I want to log in securely so that my access is protected | Login validated securely | High |
| US-009 | As a system admin, I want data encrypted so that security is ensured | AES-256 encryption applied | High |
| US-010 | As a user, I want a dashboard so that I can view asset status easily | Dashboard displays summary | Medium |
