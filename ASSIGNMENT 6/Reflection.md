# Reflection: Agile Planning Challenges

Applying Agile methodology to the Warehouse IT Asset Tracking System introduced several challenges, particularly in prioritization, estimation, and aligning requirements with user stories.

One of the main challenges was breaking down system requirements into user stories that follow the INVEST principles. While some requirements were straightforward, others needed to be simplified into smaller, testable units. Ensuring that each user story delivered value while remaining independent required careful thought.

Prioritization was another challenge. Deciding which features should be classified as Must-have, Should-have, or Could-have required balancing immediate system needs with long-term goals. For example, features like asset registration and search were clearly essential, while dashboard and reporting features were important but not critical for the initial version.

Estimating effort using story points also required judgment. Without actual development experience for the system, estimating complexity was based on assumptions. This made it challenging to assign accurate story points while still maintaining realistic planning.

Sprint planning required selecting a manageable number of stories that could realistically be completed within two weeks. The focus was placed on delivering a working MVP that includes core functionality such as login, asset registration, search, and fault reporting.

Another challenge was aligning Agile practices with the fact that this is an individual project. In real-world scenarios, Agile involves collaboration between multiple team members. In this case, all roles such as product owner, developer, and tester had to be considered individually, which required a broader perspective.

Overall, this assignment demonstrated the importance of structured planning in Agile development. It highlighted how breaking down work into smaller tasks improves clarity, while prioritization ensures that the most valuable features are delivered first. The experience also reinforced the need for flexibility, as requirements and priorities may evolve over time.
