## Prioritization Justification

Must-have features are critical for the system to function and meet stakeholder needs such as tracking assets and reporting faults.

Should-have features improve usability and maintenance but are not required for the MVP.

Could-have features enhance user experience but can be implemented later.

This prioritization ensures the MVP delivers core value while allowing future scalability.
