## 2 Non-Functional Requirements

### Usability
The system interface shall be easy to use and accessible through a web browser.

Measurement:
- New users should be able to learn basic functions within 10 minutes.

---

### Deployability
The system shall be deployable on Linux and Windows servers.

Measurement:
- Deployment must take less than 30 minutes.

---

### Maintainability
The system shall include clear documentation for developers.

Measurement:
- Documentation must include API guides and architecture diagrams.

---

### Scalability
The system shall support at least 1000 concurrent users.

Measurement:
- Performance must remain stable under peak load.

---

### Security
All user authentication data shall be encrypted.

Measurement:
- Encryption standard AES-256 must be used.

---

### Performance
System responses shall occur quickly.

Measurement:
- Asset searches must complete within 2 seconds.

---

### Availability
The system shall maintain high uptime.

Measurement:
- System availability must be at least 99%.

---

### Reliability
The system shall prevent data loss.

Measurement:
- Automatic backups must occur daily.
