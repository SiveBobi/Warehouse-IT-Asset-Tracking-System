# Reflection: Use Cases and Testing Challenges

Developing use case models and test cases for the Warehouse IT Asset Tracking System presented several challenges. One major challenge was ensuring that all use cases aligned correctly with the functional requirements defined in the previous assignment. It required careful tracing between stakeholder needs, system functionality, and user interactions.

Another challenge was balancing simplicity and completeness in the use case diagram. While it is important to include all relevant actors and use cases, overcrowding the diagram can make it difficult to understand. Therefore, only the most critical interactions were included while ensuring coverage of all major system functions.

Writing detailed use case specifications also required precision. Each use case needed clearly defined preconditions, postconditions, and flows. It was sometimes difficult to anticipate all possible alternative flows, especially error scenarios such as invalid inputs or missing data.

Test case development introduced another layer of complexity. Functional test cases had to directly map to requirements, ensuring traceability. Additionally, defining non-functional test scenarios such as performance and security required thinking beyond normal system interactions and considering system behavior under stress or attack.

Overall, this assignment highlighted the importance of structured system design. Translating requirements into use cases and test cases ensures that the system is both usable and testable. It also reinforces the importance of aligning technical implementation with stakeholder expectations.
